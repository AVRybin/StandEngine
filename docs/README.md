# Документация Stands Engine

Stands Engine разделяет инфраструктуру на три уровня:

1. **Приложение** — переиспользуемое описание образа, ролей и шаблонов запуска.
2. **Стенд** — конкретный набор серверов, инстансов приложений и их размещение.
3. **Выполнение** — настройка провайдера, создание серверов, запуск и удаление стенда.

Для каждого уровня есть отдельное руководство:

| Задача | Руководство |
|---|---|
| Добавить новое приложение в app registry | [Описание приложений](application-manifest.md) |
| Собрать или изменить `stand.yml` | [Манифест стенда](stand-manifest.md) |
| Настроить окружение и выполнить `create`/`destroy` | [Настройка и эксплуатация](operations.md) |

## Рекомендуемый порядок чтения

Если вы впервые работаете с проектом:

1. Прочитайте общий обзор и быстрый старт в [README проекта](../README.md).
2. Изучите [манифест стенда](stand-manifest.md), чтобы понять связи между
   серверами, приложениями и инстансами.
3. Если нужно добавить новый тип приложения, перейдите к
   [описанию приложений](application-manifest.md).
4. Перед реальным запуском проверьте требования в
   [руководстве по эксплуатации](operations.md).

## Примеры

- [`demo/stand.yml`](../demo/stand.yml) — полный стенд с несколькими приложениями
  и нодами.
- [`demo/app-registry`](../demo/app-registry) — готовые описания Redis, MongoDB,
  Redpanda, Kafka UI и Dozzle.
- [`demo/cloud-init.yaml.mako`](../demo/cloud-init.yaml.mako) — cloud-init для
  поддерживаемого Podman runtime.

> `create` создаёт реальные облачные ресурсы. Перед запуском проверьте provider
> credentials, типы серверов, сеть и стоимость ресурсов.
